Summary

_n.EnvironmentComparer allows to compare data between two distinct environments.
His main purpose is to simplify the migration of configuration data during the deployment process, providing a winmerge like UI.

Please note: this tool is not suitable to compare large amount of data.
If you need to compare large amount of data we suggest to apply different approaches such as copying data to staging tables on Azure SQL and apply comparison via Stored Procedures.

Steps

  1. Connect to the first environment (ENV1).
  2. Connect to a second environment (ENV2).
  3. Create (or open) a comparison map, deciding which tables/columns to compare and which rules to apply.
  4. Execute the comparison and analyze results.
  5. Track and apply actions on (copy record from ENV1 to ENV2 or vice-versa).

Connect to an environment

The plugin toolbar highlights the environments that will be used on the comparison.

  • If not connectect at startup, you can use XrmToolBox connection control to open the connection dialog and select the environment. The selected environment will be shown in the plugin toolbar, replacing the label 1. Connect to environment 1.
  • Once connected to the first environment, you can use the toolbar button 2. Connect to environment 2 to open the connection dialog and select the second environment.
    To change connection you can click again on the same button. A prompt will ask for confirmation before removing the current connection.

Please note: changing one of the connections will reset the plugin context, removing any previously executed result.
