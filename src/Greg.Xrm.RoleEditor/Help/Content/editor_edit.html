Editing capabilities

Editing capabilities are the killing feature of this plugin. They took inspiration from the legacy Dynamics CRM Role Editor, but they are much more powerful.

Privileges are divided into 2 separate tabs: Table related privileges and Miscellaneous privileges.
Table related privileges are the ones that are related to a specific table, like Read Account or Write Contact.
Miscellaneous privileges are the ones that are not related to a specific table, like Export to Excel or Act on behalf of another user.

By default:

  • If you're creating a new role, all privileges are shown.
  • If you're opening an existing role, only assigned privileges are displayed in the grid.

You can use the Show all privileges button and Show only assigned privileges button to toggle between the two views.

Table related privileges

The table related privileges tab contains a grid with a row for each table of the system, and a column for each privilege that can be assigned to that given tab.
Empty cells represent privileges that cannot be assigned for a given table (e.g. Assign or Share for the standard Currency entity).

Tabs are grouped in categories, that are freely collapsable/expandable. The list of categories, and the table-category relationship, can be freely customized in the tool Settings dialog.
You can also use the Search table capability to filter the list of tables by display name or logical name (case insensitive).

To change the privilege assigned to a table, you can simply click on the cell corresponding to the table and the privilege you want to assign.
The privilege level will be automatically increased by one level each time you click on the cell.
Only valid privilege levels are taken into account, so you can't assign a privilege level that's not valid for the specified privilege (e.g. for organization owned tables, only None or Organization can be assigned).

Modified cells are highlighted in yellow:

If you think that this way of setting privilege requires too many clicks, you can simply right click on a privilege, and a context menu will appear with all the possible privilege levels for that privilege, where you can peek what you want.

You can also select a whole row and press Space to increase all the privileges of the selected table by one level.
This works also for multiple rows selection, that can be done pressing the Shift key (in case of contiguous lines) or Ctrl key (in case of not contiguous lines).

You can also click on a column header to increase all levels of the selected privilege by one.
Please note: if you have filtered the list of tables, only the visible rows will be affected.

A basic use case is that you have configured the privileges for a table, and you need to apply the same configuration to X other tables. This is a quite common scenario when you have a set of tables that share the same structure and you want to apply the same security configuration to all of them. Using the standard UX is quite a pain in this case because you have to apply the same privileges to each table one by one.

_n.RoleEditor tool streamlines this operation in the most basic way possible, via Ctrl+C/Ctrl+V you can easily copy the privileges from one table to another, or even to a set of tables.
Of course access levels will be pasted only if valid for the target privilege.

Once modified, privileges can be saved by clicking on the "Save" button.

Miscellaneous privileges

The miscellaneous privileges tab contains all the privileges that are not related to a specific table. It presents the same structure and capabilitiesof the Table related privileges such as:

  • Privileges grouped in categories (customizable via Settings dialog)
  • Collapsable/expandable groups
  • Filtering by privilege display or technical name
  • Single or multiple edit
  • Edit by column affecting only visibile cells
  • Copy/paste (via Ctrl+C/Ctrl+V) to apply specific privilege configuration to one or more rows
