Editing roles in bulk

The Role Editor can be also used to edit up to 10 roles at once.

The structure of the Bulk Role Editor is the same of the Role Editor, but with some small difference:

  • The role editing grids (both table and misc) are by default blank.
  • Cells corrisponding to invalid privileges are greyed out.
  • When you set a given privilege (you can leverage the same capabilities of the default role editor), the cell you've updated will be highlighted in yellow.
  • Export in excel or markdown is not available.
  • Import from excel is not available.
  • You cannot save a privilege snippet from the bulk role editor (but you can apply a previously saved snippet).
  • Tab title won't show the "dirty" *

Change detection

The change detection mechanism is quite different from the one in the Role Editor. By clicking on the Change Summary tab you can see a preview of how the changes you've made will affect the roles you've selected. Some role may not be affected at all if the privilege you've set has already that specified level on them.

The result of the change detection is shown as a tree:

  • Role name
    • Operation type (Add, Remove, Replace)
      • Operation details

When you click on the Save button, only roles having actual changes are saved, and all the changes of a given role are applied transactionally.

Please note: the save operation may take a while, basing on the number of roles changed and the number of privileges affected. Do not close the window while save is in progress.
