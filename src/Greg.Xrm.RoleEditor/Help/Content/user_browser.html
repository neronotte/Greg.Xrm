User Browser

The user browser provides a visual representation of the system business unit and users hierarchy. It allows you to view the users in their hierarchical organization, and the roles assigned to each one of them.

The data in this panel are loaded automatically when the user clicks on the Load tables, privileges and roles button. The screenshot below shows a sample visual representation of the user hierarchy in a sample CDX environment.

Key elements

  • The view presents the list of environments currently loaded, and for each environment
    • The business unit hierarchy. On each busines unit
      • Any child business unit, if present
      • The list of Business Users
      • The list of Technical Users (app registrations)
  • Expanding an user you can get the list of roles assigned to that user (with evidence of the Business Unit of the given role, really useful if the Matrix data access structure is enabled.
  • You can search the user list using the Search user text box
  • You can double click a role name to open the Role Editor

Change the business unit of one or more users

This view allows you to change the business unit of one or more users using a simple drag&drop operation.

Select the user you want to move via left click. You can also use the CTRL key to select multiple users. Then, drag the selected users and drop them on the target business unit.

Please note: there are a few simple rules to keep in mind:

  • You can select users belonging to different business units but not different environments
  • If you have selected multiple users, the target business unit must not contain any of the selected users
  • The target business unit must belong to the same environment containing the selected users
  • The operation will remove all roles from the selected users. A confirm dialog will appear to ensure you have performed all the required checks before running the switch.
