Role Browser

The role browser shows a visual representation of the roles in the system, organized in the Business Unit hierarchy as they are modeled into your dataverse environment. You can load the roles for the current environment by clicking the "Load tables, privileges and roles" button of the main window.

Roles will be displayed in a tree, where the root node is the organization and the children nodes are the business units. You can change XrmToolbox connection while the tool it's opened and, by clicking the same button, you can load the roles of the new environment, that will be displayed under a new root node. This feature is really useful if you want to compare roles between different environments.

Main features

You can use the Search bar to filter the roles by name. The search is case insensitive and will match any role that contains the search term.

By default, only customizable roles are displayed. You can show all roles by clicking on the Hide not customizable roles toggle menu.

If you want to hide the roles belonging to managed solutions, you can do so by clicking on the Hide managed roles toggle menu.

You can expand and collapse the nodes of the tree by clicking on the plus and minus icons.

Double-clicking on a role will open the Role Editor window, where you can see the role configuration and edit it (if the role is customizable).

Create new roles

From the role browser, you can create new roles by clicking on the Create new role button. This will open the Role Editor window, where you can configure the new role. There are 4 ways to create a new role:

  • Start from an empty role: this will create a new role with no privileges or access levels. If you have more than one environment loaded, the new role will be created in the environemnt where the currently selected role belongs to. If no role is selected, the first environment in the list is used.
  • Clone selected role: this will create a new role with the same privileges and access levels as the selected role. The new role will be created in the environemnt where the currently selected role belongs to.
  • From "Basic User": this will create a new role with the same privileges and access levels as the "Basic User" role. The new role will be created in the environemnt where the currently selected role belongs to.
  • From "App Opener": this will create a new role with the same privileges and access levels as the "App Opener" role. The new role will be created in the environemnt where the currently selected role belongs to.
