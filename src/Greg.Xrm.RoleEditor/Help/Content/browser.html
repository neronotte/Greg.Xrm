Role Browser

The role browser provides a visual representation of the roles in the system, organized within the Business Unit hierarchy as modeled in your Dataverse environment. You can load the roles for the current environment by clicking the "Load tables, privileges, and roles" button in the main window.

Roles will be displayed in a tree structure, where the root node represents the organization, the child nodes represent the business units, and the leaf nodes are the actual roles. You can change the XrmToolbox connection while the tool is open and, by clicking the same button, load the roles of the new environment, which will be displayed under a new root node. This feature is particularly useful if you want to compare roles between different environments.

Main Features

You can use the Search bar to filter roles by name. The search is case-insensitive and will match any role containing the search term.
By default, only customizable roles are displayed. You can show all roles by clicking on the Hide non-customizable roles toggle menu.
If you want to hide roles belonging to managed solutions, you can do so by clicking on the Hide managed roles toggle menu.

You can expand and collapse the nodes of the tree by clicking on the plus and minus icons.
Double-clicking a role will open the Role Editor window, where you can view and edit the role configuration (if the role is customizable).

Create New Roles

From the role browser, you can create new roles by clicking on the Create new role button. This opens the Role Editor window, where you can configure the new role.
There are four ways to create a new role:

  • Start from an empty role: This option creates a new role with no privileges or access levels. If you have more than one environment loaded, the new role will be created in the environment of the currently selected role. If no role is selected, the first environment in the list is used.
  • Clone selected role: This option creates a new role with the same privileges and access levels as the selected role. The new role will be created in the environment of the currently selected role.
  • From "Basic User": This option creates a new role with the same privileges and access levels as the "Basic User" role (if present in the selected environment). The new role will be created in the environment of the currently selected role.
  • From "App Opener": This option creates a new role with the same privileges and access levels as the "App Opener" role (if present in the selected environment). The new role will be created in the environment of the currently selected role.
