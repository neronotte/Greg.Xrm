Assign roles to users

With this feature you can assign roles to users in bulk. You can select multiple users and assign them the same set of roles. The tool automatically provides double checks on roles already assigned to the selected users to avoid errors during the assignment process.

The feature is designed to properly manage also the scenario where Record ownership across Business Units is enabled on the environment: in this case, in fact, the same role can be assigned to an user twice but in different business units.

The tool is designed to be used in a very simple way:

  1. Select the users you want to assign roles to.
  2. Select the roles you want to assign to the selected users.
  3. Click on the Apply button.

To select the user(s) and role(s) you can:

  • Drag the users from the User Browser and drop them in the current window.
  • Drag the roles from the Role Browser and drop them in the current window.
  • Copy the names of the users and roles you want to manage, and paste them in the current window.

Drag & Drop

It's the easiest way to select users and roles. Just open the User Browser and/or the Role Browser, select the users/roles you want to configure and drag them in the current window.

You can drop them anywhere in the list component, the tool will arrange them properly.

Add users/roles via smart paste

This feature is designed to address for a real life usage scenario where the list of users/roles to configure is provided via Excel files or email.

You can simply copy a piece of text made by:

  • Users full name: e.g. Riccardo Gregori
  • Users domain name: e.g. riccardo.gregori@neronotte.onmicrosoft.com
  • Role name: e.g. System Administrator
  • Role name + Business unit name (separated by ':') : e.g. Role A:Business Unit B (only if Record ownership across Business Units is enabled)

The text can contain one or more of the above items, separated by new lines (\r\n), commas (,), semicolon (;) or pipe (|). Spaces and empty lines are ignored.

The tool will automatically parse the pasted content, extract the relevant information, match it against the actual users and roles in the system, and configure the window for the requested assignment.

For example, you can copy the following text: 

Luca Gregori
System Administrator
Test Role:Business Unit A.1
Test Role:Business Unit B

You can check the results of the assign operation in the Output window at the bottom of the screen.

In the above example, Test Role on Business Unit B has not been extracted from the text because Test Role was a role created directly under Business Unit A, so it was only available for Business Unit A and all its children (Business Unit A.1, A.2, A.3), not for Business Unit B.

Record ownership across business units

If Record ownership across Business Units is enabled, you can assign the same role twice to a given user, but on different business units.

Once selected the role(s) you want to add, the tool allows you to change the related business unit by clicking on the business unit name.

Remove user(s) and/or role(s) from selection

If by mistake you added one or more users/roles to the current window and you want to remove them, simply:

  • Select them (via LEFT CLICK, or by using CTRL+LEFT CLICK for multiple selection
  • Press the DELETE key
  • Press Yes in the confirm dialog
