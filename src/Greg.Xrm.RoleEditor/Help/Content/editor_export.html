Export in Excel or Markdown

_n.RoleEditor allows you to export the configuration of a given role in Excel or Markdown format. This capability may be useful for documentation purposes or to share the configuration with other team members.

Export can be done by clicking on one of the two "Export" buttons on the Role Editor toolbar:

The export will reflect also any unsaved changes made to the role, so make sure to save your changes before exporting.

Export in Excel

This export mode generates an Excel (xlsx) file composed by 3 sheets:

  1. General: containing the role name and a few info such as the role Business Unit, the role description and the inheritance type.
    2. .
  2. Tables: containing an Excel table with the list of all Dataverse table related privileges, grouped by table. Even unassinged privileges are shown. You can easily highlight only the tables having at least one privilege by filtering using the "Has privileges?" column.
  3. Misc: containing an Excel table with the list of all non-table related privileges, grouped in the same way they are grouped in the role editor. For each privilege, also the technical name is shown in a dedicated column.

Export in Markdown

Markdown is the official format used by GitHub or Azure DevOps to render documentation. This export mode generates a Markdown file with a structure similar to the Excel export, but in a plain text format.

In the exported file, privileges are represented using emojis, for an easy and quick visual recognition.

The main difference with the excel export is that only the tables having at least one privilege are shown, and only the miscellaneous privileges actually assigned to the role are listed.
